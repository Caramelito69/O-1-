¿QUÉ HACE UNA
COMPUTADORA?

NO piensa

!

// Ejemplo · Minecraft
// ¿El jugador puede construir un pico?

// ❌ Incompleto
si (madera) {
// solo verifica madera, olvida piedra }

// ✅ Correcto
si (madera Y piedra) {
construir("pico")
} fin si

// DECISIÓN SIMPLE
Estructura básica · Decisión simple

ESTRUCTURA
SI

Se ejecuta solo si la condición es verdadera. Si es falsa, no pasa nada y el programa continúa.
Evalúa la condición
Si es verdadera → ejecuta el bloque
Si es falsa → no hace nada
Pseudocódigo · SI simple
// Sintaxis
si (condicion) {
ejecucion
} fin si

// Ejemplo
nivel = 15

si (nivel >= 10) {
mostrar("Puedes entrar al dungeon")
} fin si

// DOS CAMINOS
Decisión con alternativa

SI —
SINO

Ahora tenemos dos caminos. Si la condición es verdadera va por uno, si es falsa va por el otro. Siempre entra por uno de los dos.
Verdadero → ejecuta el bloque del SI
Falso → ejecuta el bloque del SINO
Pseudocódigo · SI / SINO
edad = 17

si (edad >= 18) {
mostrar("Mayor de edad")
} sino {
mostrar("Menor de edad")
} fin si

// edad = 17 → entra al SINO
// Resultado: "Menor de edad"

// MÚLTIPLES CAMINOS
Múltiples condiciones

SI — SINO SI
— SINO

El programa evalúa de arriba hacia abajo y entra solo en la primera condición verdadera. Esto es clave en competencia.
▲
Una vez que entra en un bloque, no evalúa los demás, aunque también sean verdaderos.
Pseudocódigo · SI / SINO SI / SINO
nota = 75

si (nota >= 90) {
mostrar("Excelente")
} sino si (nota >= 70) {
mostrar("Aprobado")
} sino {
mostrar("Reprobado")
} fin si

// nota = 75 → entra al "sino si"
// Resultado: "Aprobado"

// PARTE 4 · LÓGICOS
Parte 4 · Lógica combinada · 25 min

OPERADORES
LÓGICOS

Y  (AND)
Todas las condiciones deben cumplirse.
si (energia >= 50
   Y espada == verdadero) {
atacar()
} fin si
O  (OR)
Al menos una debe cumplirse.
si (llave_oro
   O llave_maestra) {
abrir_puerta()
} fin si
NO (NOT)
Invierte la condición.
si (NO muerto) {
seguir_jugando()
} fin si

// PRECEDENCIA
¡Muy importante!

PRECEDENCIA
LÓGICA

Cuando combinas operadores, el orden de evaluación importa. Se evalúan en este orden:
1° · NO
2° · Y
3° · O
★
Consejo profesional: Usa paréntesis SIEMPRE para evitar errores. No dependas de la precedencia.
Precedencia · Ejemplo
// Sin paréntesis — peligroso
si (a O b Y c) {…}

// Se evalúa como:
si (a O (b Y c)) {…}

// ¿Querías esto? Quizás no.

// ✅ Con paréntesis — claro y seguro
si ((a O b) Y c) {
ejecucion
} fin si

// PROBLEMA 3
Problema real competitivo · Nivel avanzado

NÚMERO
ESPECIAL

Un número es especial si cumple las 3 condiciones a la vez:
es positivo
es par (divisible entre 2)
es múltiplo de 3
%
El operador MOD (residuo) nos dice si un número es divisible. Si n MOD 2 == 0 → el número es par.
Solución · Pseudocódigo
n ← leer()

si (n > 0
   Y n MOD 2 == 0
   Y n MOD 3 == 0) {
mostrar("Especial")
} sino {
mostrar("Normal")
} fin si

// n=6 → positivo, par, mult3 → ✅
// n=4 → positivo, par, NO mult3 → ❌
// n=0 → NO positivo → ❌

MINI RETO
FINAL

Un jugador sobrevive si:
tiene comida Y agua
O tiene un kit de supervivencia
!
Presten atención a los paréntesis. (comida Y agua) se evalúa primero, luego el O kit.
Solución esperada
comida ← verdadero
agua ← falso
kit ← verdadero

si ((comida Y agua) O kit) {
mostrar("¡Sobrevive!")
} sino {
mostrar("Muere...")
} fin si

// comida=V, agua=F, kit=V
// (V Y F) O V → F O V → ✅ Sobrevive
